## Abstract
This paper presents the design, implementation, and evaluation of an offline, voice-enabled educational assistant for Nepali-speaking students. The system targets low-resource learning environments in Nepal by integrating offline automatic speech recognition (ASR), Support Vector Machine (SVM)–based intent classification, and knowledge-base retrieval into a single prototype. A custom dataset of 2,150 Nepali question–answer pairs sourced from primary school textbooks across six subject intents (Science, Social Studies, Nepali Language, Health, Physical Education, and Arts) was cleaned and used for training and evaluation. The intent classifier uses TF-IDF character n-gram features and an optimized Linear SVM pipeline (TF-IDF char n-grams + TruncatedSVD + SMOTE + GridSearchCV). On a held-out test set (n=430), the final model achieved **74.65%** accuracy with macro F1 of **77.53%**. Inference benchmarking measured a mean end-to-end (intent + retrieval) latency of **21.9 ms** per query on CPU, supporting real-time offline use. The full system includes a Tkinter GUI, microphone input with Faster-Whisper ASR, and Nepali text-to-speech via eSpeak-NG.

**Keywords** — Nepali NLP, offline voice assistant, intent classification, TF-IDF, linear SVM, SMOTE, Faster-Whisper, edge deployment.

## I. INTRODUCTION
Nepal faces a persistent gap in accessible Nepali-language educational technology, especially in rural contexts where internet access and learning resources are limited. Most mainstream voice assistants rely on cloud services and are optimized for high-resource languages, which reduces their usefulness for Nepali students requiring immediate, offline support. This project investigates whether a lightweight, classical machine learning (ML) pipeline can deliver practical intent classification for Nepali educational questions and connect that intent to an offline knowledge base for answer retrieval.

This work contributes:
- An end-to-end offline prototype for Nepali student assistance (ASR → intent classification → retrieval → TTS).
- A cleaned Nepali educational Q&A dataset (2,150 samples; six subject intents).
- An optimized Linear SVM intent classifier for Nepali educational queries using TF-IDF character n-gram features.
- Evaluation evidence including accuracy/F1, confusion matrix, and latency benchmarking artifacts generated by the provided code.

## II. THE DATA SET
### A. Problem Definition
The primary ML task is multi-class intent classification. Given a Nepali query \(q\), the classifier predicts one intent label \(y\) from:
\(\{\)arts\_question, health\_question, nepali\_language\_question, physical\_education\_question, science\_question, social\_question\(\}\).
The predicted label is then used to narrow the search space for offline answer retrieval from a local knowledge base of textbook Q&A pairs.

### B. Dataset Source and Composition
Because no suitable public Nepali educational Q&A dataset aligned to the intended subject-specific intents was available for this prototype, a custom dataset was constructed from Nepali primary school textbooks (Grades 4–7), producing 2,150 cleaned question–answer samples with intent labels.

### C. Class Distribution
The dataset contains moderate class imbalance (Science and Social Studies dominate). Table 1 summarizes sample counts.

## III. DATA PREPARATION
### A. Text Normalization
The dataset is cleaned using a Devanagari-aware normalization routine (implemented in `codes/data_prep.py`), including:
- Removal of leading Nepali/ASCII numbering patterns.
- Removal of repeated instructional noise phrases (e.g., “लेख्नुहोस्”, “बताउनुहोस्”).
- Dropping null/short questions and standardizing column names.

### B. Train/Test Split
An 80/20 stratified split is used with fixed random seed (random\_state=42), resulting in 1,720 training and 430 test samples. Stratification preserves class proportions across splits.

### C. Feature Representation
All models use TF-IDF character word-boundary n-grams:
- analyzer=`char_wb`
- ngram\_range=(2,4)
- sublinear\_tf=True

Character n-grams are selected to be robust to Nepali morphology, spelling variation, and ASR noise.

### D. Class Imbalance Handling
Two complementary strategies address imbalance:
- `class_weight='balanced'` for linear classifiers.
- SMOTE oversampling within the training pipeline for the optimized SVM model (applied inside cross-validation folds to avoid leakage).

---

## IV. MACHINE LEARNING CLASSIFICATION TECHNIQUES
This project focuses on a single model family for the intent classification task: a **Linear Support Vector Machine (LinearSVC)** trained on TF-IDF character n-gram features. Linear SVM is effective for text classification due to maximum-margin separation in high-dimensional spaces and typically performs well in low-resource NLP settings.

### A. Linear Support Vector Machine (SVM)
The optimized pipeline uses:
- TruncatedSVD (dimensionality reduction; enables dense-space SMOTE)
- SMOTE oversampling
- GridSearchCV with 5-fold stratified CV to tune C and n-gram range

## V. APPLICATION OF TECHNIQUES AND RESULTS
### A. Implementation and Prototype
The prototype is implemented in Python and includes:
- Offline ASR using Faster-Whisper (`Systran/faster-whisper`) with CPU INT8 inference.
- Intent classification using the trained model (best: optimized Linear SVM).
- Offline answer retrieval via TF-IDF cosine similarity restricted by predicted intent.
- Offline TTS using eSpeak-NG (fallback to pyttsx3).
- A Tkinter GUI (`codes/app.py`) supporting typed and microphone input.

### B. Overall Results (Optimized SVM)
The final intent classifier is an optimized Linear SVM model. Evaluation metrics are reported on a held-out test set (n=430) and recorded in `outputs/eval_report.json`:
- Accuracy: **74.65%**
- Macro F1: **77.53%**
- Weighted F1: **74.64%**

## VI. CONCLUSION
This work demonstrates that an offline Nepali student assistant can be implemented with classical ML methods while remaining fast and practical on CPU-only hardware. The optimized Linear SVM pipeline achieved 74.65% test accuracy with 77.53% macro F1 and maintained low per-query latency (21.9 ms mean for intent + retrieval). The results support feasibility for low-resource immediate systems and establish a reproducible baseline for Nepali educational intent classification using SVM.

## VII. FUTURE RESEARCH
Future improvements include:
- Expanding the dataset (more grades, more intents, more diverse phrasing).
- Formal ASR evaluation (e.g., Word Error Rate on a Nepali speech test set) and robustness testing under classroom noise.
- Further optimization of the SVM pipeline (e.g., additional feature experiments, calibration, thresholding for “unknown” intent).
- Running user studies with Nepali students/teachers to assess learning impact and usability.
- Improving retrieval with BM25 or hybrid semantic retrieval (if fully offline embeddings are feasible).
