Abstract—This paper presents a fully offline voice-enabled educa-
tional assistant for Nepali-speaking students, addressing the shortage of
accessible Nepali-language learning tools in low-resource settings. The
system combines Automatic Speech Recognition (ASR) using Faster-
Whisper, Devanagari-aware text normalization, intent classification,
and intent-filtered answer retrieval into a single local application
with a Tkinter GUI and eSpeak-NG offline text-to-speech. A custom
dataset of 2,150 question-answer pairs was constructed from Nepali
primary school textbooks (Grades 4–5) covering Science, Social
Studies, Nepali Language, Health, Physical Education, and Arts. Three
classical machine learning pipelines were systematically compared,
each using TF-IDF character n-gram features, TruncatedSVD dimen-
sionality reduction, and SMOTE oversampling. Logistic Regression
achieved the highest performance with 75.12% test accuracy and
78.12% macro F1, while Linear SVM scored 74.65% and 77.53%
respectively. Despite marginally lower scores, SVM was selected for
deployment due to its interpretability through linear weight vectors
and faster inference. Hyperparameter analysis revealed that C=1 with
ngram_range=(2,4) is the critical configuration, with 4-character
sequences capturing Devanagari morphological patterns missed by
shorter n-grams. Feature interpretability analysis identified the most
discriminative character n-grams per intent class. The end-to-end
pipeline achieved an average latency of 21.3 ms per query on CPU-only
hardware, confirming suitability for Raspberry Pi 4 edge deployment.
The complete system is supported by fully reproducible scripts.
Index Terms—Nepali NLP, offline voice assistant, intent classifi-
cation, TF-IDF, Logistic Regression, Linear SVM, Faster-Whisper,
low-resource education.
I. INTRODUCTION
Nepal presents a challenge for educational technology: with
internet penetration below 40% in rural areas [ 2], cloud-
based voice assistants remain inaccessible to most students [ 3].
Although primary students encounter questions across Science,
Social Studies, Nepali Language, Health, Physical Education,
and Arts, language-native offline tools remain scarce.
This work investigates whether a classical machine learning
pipeline can support practical Nepali educational question
answering entirely offline. The prototype combines four stages:
Faster-Whisper ASR, Devanagari-aware text normalization,
intent classification, and TF-IDF cosine retrieval over a fixed
knowledge base. The system runs on CPU-only hardware
including Raspberry Pi 4, with a Tkinter GUI and eSpeak-
NG speech output.
The main contributions are:•A custom 2,150-sample Nepali educational Q&A dataset
across six intent classes, built from scratch.
•A comparative evaluation of three classical classifiers
(Logistic Regression, Linear SVM, KNN) with shared
preprocessing and SMOTE.
•Hyperparameter sensitivity analysis via 5-fold stratified
GridSearchCV .
•Feature interpretability analysis identifying discriminative
Devanagari character n-grams per intent class.
•A deployable offline prototype integrating ASR, clas-
sification, retrieval, and TTS in a single reproducible
application.
•Discussion of social, ethical, legal, and professional
implications.
The remainder of the paper is organized as follows. Sec-

Section IV presents the methods. Section V covers the
experimental setup. Section VI reports results. Section VII
discusses implications and limitations. Section VIII concludes.
II. LITERATUREREVIEW
A. NLP for Low-Resource Languages
Natural Language Processing for low-resource languages
has advanced significantly through multilingual transformer
models such as mBERT [ 5] and XLM-R [ 6]. However, these
models typically require greater memory and computation
than is practical for low-cost offline deployment. Classical
text classification methods based on TF-IDF features and
linear classifiers remain attractive when hardware constraints,
reproducibility, and interpretability matter [7].
For Devanagari-script languages, character-level n-grams are
especially useful because they are robust to spelling variation,
inflection, and tokenization noise. Sharma et al. [ 8] reported
that character-level representations can outperform word-level
features on related script-classification tasks, which motivates
their use in this project.
B. Intent Classification for Educational Systems
Intent classification is central to task-oriented dialogue
systems because it maps free-form user input into an actionable
2
semantic category [ 9]. In educational support systems, classical
supervised methods can still perform strongly when the number
of classes is limited and the domain is well-scoped [ 10]. This
is a good fit for the present problem, where the assistant only
needs to classify student questions into a small set of subject
areas before retrieving a relevant answer.
C. Speech Recognition for Nepali
Nepali ASR remains relatively under-resourced compared
with English and other major languages. Whisper-style multilin-
gual ASR models are among the first broadly usable systems for
Nepali transcription [ 11]. Faster-Whisper, built on CTranslate2,
makes local CPU inference more practical through optimized
and quantized inference, which is why it was selected for
this prototype [ 4]. Earlier Nepali ASR work, such as CMU
Sphinx-based systems, showed feasibility but with more limited
robustness and harder deployment pipelines [12].
D. Retrieval in Restricted Knowledge Domains
When the response space is restricted to a fixed textbook-
derived knowledge base, classical sparse retrieval remains a
strong baseline. TF-IDF cosine similarity is easy to implement,
explain, and deploy offline, and it avoids the complexity of
dense embedding systems. Intent-conditioned retrieval further
reduces false matches by narrowing the search to the predicted
subject category[13], [14].

III. PROBLEM ANDDATASETDESCRIPTION
A. Problem Formulation
The primary machine learning task is multi-class intent
classification. Given a Nepali question q, obtained either from
typed input or from ASR transcription, the system predicts
one of six intent labels: science_question ,social_
question ,nepali_language_question ,health_
question ,physical_education_question , and
arts_question . The predicted intent is then used to
constrain retrieval over the local knowledge base so that the
answer search is performed only within the relevant subject
area.
B. Dataset Construction
Existing public repositories including Kaggle, the UCI ML
Repository, and HuggingFace Datasets were searched for
Nepali educational Q&A datasets; none matching the required
domain specificity or Devanagari script format were found. A
custom dataset was therefore constructed from Nepali primary
school textbooks (Grades 4–5) [ 16] covering the six target
subject domains. Questions and answers were extracted fromPDF sources using a Python pipeline ( pdfplumber with
custom Devanagari regex), followed by manual annotation
of intent labels. The raw collection yielded 2,214 records;
after preprocessing (described in Section V), 64 malformed
or ambiguous records were removed, yielding 2,150 usable
samples.
The distribution is moderately imbalanced. Science and
Social Studies dominate the dataset, while Arts and Physi-
cal Education are minority classes. This imbalance directly
influenced the training design, particularly the use of SMOTE
and balanced linear models.
C. Data Fields and Significance
Each row in the cleaned spreadsheet includes: question ,
answer ,intent ,grade , andtopic . The question and
intent fields are required for training, while answer is used
later in the retrieval stage.
Beyond the classification task itself, an offline Nepali
assistant can support student self-study, reduce dependence
on internet connectivity, and make subject-specific help more
available in schools and homes with limited digital infrastruc-
ture.
IV. SYSTEM ANDMETHODS
A. End-to-End System Overview
The implemented assistant follows a four-stage pipeline:
1) Speech or typed input is captured from the user.
2)Faster-Whisper converts Nepali speech to text when
microphone mode is used.
3)The text is normalized and classified into an intent label.
4)A TF-IDF retrieval index returns the most relevant answer
within the predicted intent class.

retrieval and speech output.

The preprocessing pipeline is implemented in
codes/data_prep.py . The function normalize_
nepali() removes leading question numbers written in
either ASCII digits or Devanagari digits and strips common
instructional prefixes that are not semantically useful for
classification. The cleaned data is then filtered to remove rows
with missing question orintent values and very short
malformed questions.

The retrieval stage is implemented in codes/build_
kb.py and in the runtime fallback inside codes/app.py .
A TF-IDF vectorizer with char_wb analyzer and ngram_
range=(2,4) is fitted on the stored questions. At inference
time, the input question is vectorized, filtered by predicted
intent, and matched against the corresponding knowledge-base
entries using cosine similarity. The best-scoring answer above
the configured threshold is returned.

The speech front-end is implemented in codes/voice_
engine.py . Faster-Whisper is configured for Nepali using
thebase model with CPU execution and INT8 quantization.

Three classical machine learning pipelines were evaluated:

over: tfidf__ngram_range∈ {(2,3),(2,4)} ;svd_

V. EXPERIMENTALSETUP
A. Data Preprocessing and EDA
Raw data was loaded from question_answer_final_
updated.xlsx . The normalize_nepali() function
performed: (1) stripping leading Devanagari and ASCII serial
numbers; (2) removing instructional noise words; (3) column-
name normalization; (4) missing grade imputation via nearest-
neighbour interpolation; and (5) dropping records with null
question orintent and questions shorter than three char-
acters. Of 2,214 raw records, 64 were removed as malformed,
yielding 2,150 usable samples.
Fig. 2. Intent distribution in the cleaned dataset (2,150 samples). Science
and Social Studies represent ≈60.8% of records combined, while Arts and
Physical Education together account for only ≈6.6%, motivating SMOTE and
class-aware evaluation.
B. Train/Test Split
A stratified 80/20 train-test split was applied ( random_
state=42 ), yielding 1,720 training samples and 430 test
samples. Stratification ensures proportional class representation
in both partitions, which is critical given the 10:1 imbalance
between Science (686 samples) and Arts (64 samples).C. Cross-Validation
Five-fold stratified cross-validation was used within
GridSearchCV on the training set, optimizing weighted F1-
score. Stratification inside each fold preserves the original
class ratios, providing a robust generalization estimate across
all hyperparameter combinations without touching the held-out
test set.

E. Evaluation Metrics
The following metrics were used: Accuracy (proportion
of correctly classified test samples), Macro F1 (unweighted
mean F1 across all six classes, sensitive to minority-class
performance), Weighted F1 (F1 weighted by class support),
Confusion Matrix (per-class heatmap revealing error patterns),
and Latency (wall-clock time in ms per query over 50 held-out
queries usingperf_counter).

G. Feature Interpretability
A key advantage of Linear SVM is the ability to inspect
the learned weight vectors. After training, the SVM coefficient
matrix was projected back to the original TF-IDF feature
space, and the top-5 most discriminative character n-grams
were extracted per intent class. The analysis revealed that
science_question is driven by character patterns cor-
responding to Nepali scientific terminology (e.g., syllables
from Devanagari words forlightandenergy), nepali_
language_question is distinguished by literary and gram-
matical morphemes, and arts_question features unique
creative-domain vocabulary not shared with other classes.
H. Interpretation of Findings
The results support three practical conclusions. First, classi-
cal machine learning is sufficient to make a Nepali educational
assistant viable without large neural language models. Second,
character-level features are effective for Devanagari text because
they tolerate spelling variation and short ASR-transcribed ut-
terances. Third, there is a deliberate trade-off between held-out
accuracy and deployment choice: Logistic Regression scored
marginally higher (75.12% vs. 74.65%), but the deployed SVM
is preferred for its interpretable weight vectors, comparable
speed, and established robustness in high-dimensional text
tasks [7].
VII. DISCUSSION
A. Originality and Contribution
The originality of this work lies less in inventing a new
algorithm and more in building a complete, reproducible, and
context-specific system for a low-resource educational setting.
The key contributions are a custom Nepali educational intent
dataset, a complete offline voice-assistant prototype for local
use, a comparative evaluation of multiple classical models rather
than a single-model claim, reproducible outputs for training
and evaluation, and a deployment-oriented integration of ASR,
classification, retrieval, and TTS.B. Social, Ethical, Legal, and Professional Considerations
Social impact:The assistant addresses educational inequality
in Nepal by providing an offline, Nepali-native tool aligned
with UNESCO SDG 4 (Quality Education) [ 17]. Running on
low-cost hardware like Raspberry Pi 4, it can reach students
in rural schools with critically low teacher–pupil ratios.
Ethical considerations:The retrieval-based design prevents
fabricated answers; when similarity falls below 0.25, the
assistant signals uncertainty. No personal data was collected—
all content derives from publicly distributed textbooks.
Bias and fairness:The dataset reflects standard textbook
content, which may embed cultural or gender biases. SMOTE
and balanced class weights reduce statistical imbalance but
do not eliminate source bias; future dataset auditing is recom-
mended.
Legal considerations:The system operates offline with no
data transmission. Open-source components are used under
permissive licenses (MIT, GPL v3, BSD-3-Clause). Textbook
content is used for academic research under fair-use provisions.
Professional practice:The codebase is structured
into single- responsibility modules ( data_prep.py ,
train_model.py ,build_kb.py ,evaluate_and_
benchmark.py ,app.py ). All experiments use random_
state=42 , serialized outputs, and scriptable regeneration,
ensuring reproducibility.
C. Limitations
The project also has clear limitations:
•The dataset is relatively small at 2,150 samples.
•The assistant covers only six school subject intents.
•The ASR component was integrated but not formally
evaluated on a held-out Nepali speech benchmark.
•The retrieval stage is limited to the stored knowledge base
and cannot answer unseen curriculum content.
•The deployed prototype currently uses SVM even though
Logistic Regression scored slightly higher in the latest
held-out comparison.
D. Future Work
Useful next steps include expanding the dataset with more
grade levels and question diversity, evaluating ASR formally
using a dedicated Nepali speech test set, testing whether
Logistic Regression should replace SVM as the default de-
ployed classifier, exploring lightweight transformer models
only if they can still meet offline deployment constraints, and
conducting user studies with real students and teachers to
measure usefulness beyond offline accuracy metrics.
VIII. CONCLUSION
This project demonstrated that an offline Nepali student
assistant can be built effectively using classical machine
learning and lightweight retrieval. The system integrates
Faster-Whisper ASR, Devanagari-aware preprocessing, intent
classification, TF-IDF retrieval, and optional TTS into a single
local application. On the current held-out test split, Logistic
Regression achieved the strongest classification performance at
7
75.12% accuracy and 78.12% macro F1, while the deployed
Linear SVM remained very close at 74.65% accuracy and
77.53% macro F1. The deployed SVM pipeline also achieved a
mean end-to-end latency of 21.3 ms, which supports practical
offline interaction on CPU-only hardware.
The main conclusion is that a useful Nepali educational
assistant does not require a heavyweight cloud-based model.
SVM was selected for deployment over the marginally higher-
scoring Logistic Regression for its interpretability and robust-
ness in high-dimensional text spaces. A carefully engineered
local pipeline can already provide fast, interpretable, and
reproducible support for low-resource educational settings.
